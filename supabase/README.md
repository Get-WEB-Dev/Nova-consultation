# Nova Health — Supabase Backend

## Setup

### 1. Create a Supabase project
Go to https://supabase.com/dashboard → New project

### 2. Copy environment variables
```bash
cp .env.example .env.local
# Fill in NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY
```

### 3. Run the schema
In your Supabase project → SQL Editor → New query:
1. Paste and run `supabase/migrations/001_schema.sql`
2. Paste and run `supabase/seeds/001_seed.sql`

### 4. Enable Realtime
Dashboard → Database → Replication → enable these tables:
- `consultation_queue`
- `messages`
- `doctor_profiles`
- `notifications`
- `consultations`

### 5. Create Storage buckets
Dashboard → Storage → New bucket:
- `chat-attachments` — Private, 10MB limit
- `doctor-assets` — Public

### 6. (Optional) Local dev with Supabase CLI
```bash
npx supabase login
npx supabase init
npx supabase start
npx supabase db push
npx supabase db seed
```

## Architecture

```
lib/
  supabase/
    client.ts          ← Browser + admin Supabase clients, storage helpers
    auth.ts            ← Supabase auth (replaces localStorage auth)
    database.types.ts  ← TypeScript types matching the schema

  realtime/
    subscriptions.ts   ← All Supabase realtime channel subscriptions

  server/
    queries.ts         ← Server-side data access (used by API routes)

  api.ts               ← Client-side data access (calls API routes, mock fallback)
  types.ts             ← Frontend-facing TypeScript interfaces

app/api/
  doctors/             → GET  (public, SEO)
  doctors/[id]/        → GET  (public, SEO)
  consultations/       → GET, POST
  queue/               → GET, POST, DELETE
  messages/            → GET, POST
  notifications/       → GET, PATCH
  saved-doctors/       → GET, POST, DELETE
  remind-me/           → POST, DELETE
  posts/               → GET  (public, SEO)
  video/token/         → POST (LiveKit token generation)

supabase/
  migrations/001_schema.sql   ← Full schema with RLS, indexes, triggers
  seeds/001_seed.sql          ← Test data (6 doctors, patients, consultations)
  config.toml                 ← Local dev config
```

## Realtime subscriptions

```typescript
import {
  subscribeToQueue,
  subscribeToMessages,
  subscribeToDoctorStatus,
  subscribeToNotifications,
  subscribeToConsultationStatus,
} from '@/lib/realtime/subscriptions';

// In a component:
useEffect(() => {
  const unsub = subscribeToQueue(doctorId, (rows) => setQueue(rows));
  return unsub;
}, [doctorId]);
```

## Video integration (LiveKit)

Each consultation has a unique `video_room_name` auto-generated by DB trigger:
`nova-{consultation-uuid-no-dashes}`

To generate a LiveKit token:
```
POST /api/video/token
{ consultationId, participantIdentity, participantName }
→ { token, roomName, serverUrl }
```

Replace the mock token logic in `app/api/video/token/route.ts` with:
```bash
npm install livekit-server-sdk
```
Then use `AccessToken` from the SDK (see comments in the route file).
